# Context Signals MCP v1.1 Implementation Plan

## Overview

v1.1 focuses on solving the **cold start problem** identified through real-world testing:

- Cold search can be ~10x worse than baseline
- Small projects with few queries showed +65.6% overhead
- Empty signal store causes fallback reads → double overhead

**Strategic insight:** The real bottleneck is not extraction capability but the **lifecycle of signals**: cold → warm → incremental → reusable.

## Feature Specifications

### 1. Auto-Index Bootstrap

**Goal:** Remove "must manually call signals_scan first" requirement.

**Behavior:**

```
MCP Server Starts
      │
      ▼
Check signals.json exists AND not empty?
      │
      ├── NO → Trigger background scan
      │
      └── YES → Check index-meta.json exists?
                │
                ├── NO → Trigger full scan
                │
                └── YES → Check files for changes
                          │
                          ├── Changes found → Incremental update
                          │
                          └── No changes → Ready (warm cache)
```

**Startup Strategy:**
- Primary: Index on MCP server startup (non-blocking background)
- Fallback: Lazy index on first query if startup skipped/failed

**Background Indexing:**
- Does NOT block query processing
- Returns empty results OR falls back to synchronous scan
- Logs indexing progress for debugging

---

### 2. Incremental Indexing

**Goal:** Only re-index changed files, keep existing signals for unchanged files.

**Flow:**

```
For each file in project:
  │
  ├── Not in index-meta.json → NEW → INDEX
  │
  ├── mtime unchanged → SKIP (signal valid)
  │
  └── mtime changed → Compute hash
            │
            ├── hash same → SKIP (touch only)
            │
            └── hash changed → RE-INDEX

Deleted files → Remove from signals
```

**Hybrid Approach (mtime + hash):**
1. Check mtime first (fast)
2. If mtime changed, compute hash (accurate)
3. If hash same as stored → skip
4. If hash changed → re-index

---

### 3. Index Metadata

**Goal:** Track per-file change history for incremental indexing.

**File Location:** `.crush-memory/index-meta.json`

**Structure:**

```json
{
  "version": "1.1",
  "lastFullScan": "2026-04-30T12:00:00.000Z",
  "files": {
    "src/app/user/route.ts": {
      "mtime": "2026-04-30T11:30:00.000Z",
      "hash": "a1b2c3d4e5f6...",
      "indexedAt": "2026-04-30T11:30:05.000Z"
    },
    "src/utils/auth.ts": {
      "mtime": "2026-04-29T09:00:00.000Z",
      "hash": "f6e5d4c3b2a1...",
      "indexedAt": "2026-04-29T09:00:02.000Z"
    }
  }
}
```

**Operations:**
- `readIndexMeta()`: Load existing metadata
- `writeIndexMeta()`: Persist metadata after scan
- `updateFileMeta()`: Update single file entry
- `removeFileMeta()`: Remove entry for deleted files

## Appendix: Reference Implementation

### index-store.ts (Interface Reference)

```typescript
import { promises as fs } from 'fs'
import path from 'path'

const META_FILE = '.crush-memory/index-meta.json'

export interface FileMeta {
  mtime: string
  hash: string
  indexedAt: string
}

export interface IndexMeta {
  version: string
  lastFullScan: string
  files: Record<string, FileMeta>
}

export async function readIndexMeta(): Promise<IndexMeta | null> {
  try {
    const content = await fs.readFile(META_FILE, 'utf-8')
    return JSON.parse(content)
  } catch {
    return null
  }
}

export async function writeIndexMeta(meta: IndexMeta): Promise<void> {
  const dir = path.dirname(META_FILE)
  await fs.mkdir(dir, { recursive: true })
  await fs.writeFile(META_FILE, JSON.stringify(meta, null, 2))
}

export function createEmptyMeta(): IndexMeta {
  return {
    version: '1.1',
    lastFullScan: new Date().toISOString(),
    files: {}
  }
}
```

### incremental-scan.ts (Pseudo-code)

```typescript
async function incrementalScan(root, patterns, exclude, previousMeta) {
  const currentFiles = await scanner.scan(root, patterns, exclude)
  const currentMeta = createEmptyMeta()
  const filesToIndex: string[] = []
  const filesToRemove: string[] = []

  for (const file of currentFiles) {
    const prev = previousMeta?.files[file]

    if (!prev) {
      // New file
      filesToIndex.push(file)
    } else if (file.mtime === prev.mtime) {
      // Unchanged
      currentMeta.files[file] = prev
    } else {
      // mtime changed, check hash
      const hash = await computeHash(file.content)
      if (hash !== prev.hash) {
        filesToIndex.push(file)
      } else {
        // Touch only
        currentMeta.files[file] = { ...prev, indexedAt: new Date().toISOString() }
      }
    }
  }

  // Find deleted files
  if (previousMeta) {
    for (const oldFile of Object.keys(previousMeta.files)) {
      if (!currentFiles.has(oldFile)) {
        filesToRemove.push(oldFile)
      }
    }
  }

  return { filesToIndex, filesToRemove, meta: currentMeta }
}
```
